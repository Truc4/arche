# W0030 ‚Äî a proc is not a primitive

`proc` has been removed as a general callable. It is **reserved for the foreign/primitive boundary**:
a `proc` is legal only as a `#foreign` extern, a `@syscall(n)`, an `@intrinsic`, or a `@drop` destructor
hook. Every other `proc` ‚Äî pure logic, an effectful step, a helper ‚Äî is W0030, an **error by default**.

This is the load-bearing rule of the effect model: the non-foreign world has exactly three kinds, and a
"procedure" is composed from them rather than written as one.

- **`func`** ‚Äî pure value code. It produces results either as a single `-> T` return **or** as an
  out-parameter list `func(in)(out, ‚Ä¶)` (the form `proc` used to carry, for multiple results or filling a
  caller buffer). A func performs no effects.
- **`system` / `each`** ‚Äî the effect leaves and composer. A `system` runs effects (an `Eff` with `()`,
  a `#foreign` proc, `insert`/`delete`, `fmt.printf`) over ambient pool state; an `each (query{‚Ä¶})` is the
  per-row effectful fan. They take no value parameters ‚Äî they read the world's columns in place.
- **`map`** ‚Äî the pure, branch-free data-parallel kernel.

The program's entry is a **`#run`** schedule, not a `main`.

## Converting a proc

| the old proc was‚Ä¶ | becomes |
|---|---|
| pure, one result | `func(in) -> T` |
| pure, several results / fills a buffer | `func(in)(out, ‚Ä¶)` ‚Äî a func with out-params |
| runs effects, no per-row work | a `system` (reads ambient pool/singleton state) |
| runs effects per entity | an `each (query{‚Ä¶})` |
| a result-dependent sequence | decompose across systems ‚Äî a producer writes a column, a consumer reads it next |
| a foreign C call / syscall / intrinsic | stays a `proc` inside `#foreign` / under `@syscall` / `@intrinsic` |
| a `@drop` destructor | stays a `proc` with `@drop(type)` |

## Example

```arche
// BAD: a non-foreign proc ‚Äî W0030
sum_diff :: proc(a: int, b: int)(s: int, d: int) { s = a + b; d = a - b; }   // pure ‚Üí should be a func
greet    :: proc() { fmt.printf("hi\n"); }                                   // effect ‚Üí should be a system

// GOOD
sum_diff :: func(a: int, b: int)(s: int, d: int) { s = a + b; d = a - b; }   // a func owns the out-param form
greet    :: system { fmt.printf("hi\n"); }                                   // effects run in a system
#run greet
```

## Severity & escape hatch

W0030 is an **error by default**. It is still a tunable lint:

- `--proc-not-primitive=error` ‚Äî the default.
- `--proc-not-primitive=warn` ‚Äî report without failing (for code not yet migrated).
- `--proc-not-primitive=allow` ‚Äî silence it.

Or suppress a single decl with `@allow(proc_not_primitive)`. This is reserved for the rare callable that
genuinely cannot be expressed otherwise ‚Äî e.g. a **compile-time-reflective, archetype-generic** loader
(`proc(arch: archetype)` + `#each_field`), which fits neither a pure func nor an argless system. The stdlib
`csv.load` is the one sanctioned case program-wide, pending a first-class "archetype-targeted load" feature.
Do not reach for `@allow` to skip a migration.

See `docs/design/the-flat-effect-model.md` and `docs/design/callbacks-as-data.md`.
