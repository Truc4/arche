# E0121: duplicate_region

A file may carry **at most one region of each kind** — one `#module`, one `#file`, one
`#foreign`, and one `#import`. This error fires on the second region of a kind already seen
in the same file.

The two spellings of a region are the *same kind*: a `#foreign` banner and a `#foreign { … }`
block both count as the file's one `#foreign` region, and `#import io` and `#import { … }`
both count as its one `#import`. So you cannot follow one with another of the same kind.

```arche
#import { os term }
#import fmt          // error: a second `#import` region in this file

#foreign { abs :: proc(int)(int) }
#foreign             // error: a second `#foreign` region in this file
```

Fix: merge the regions into one. Collect every import into a single `#import { … }` block,
and put all foreign declarations under one `#foreign` region (banner or block):

```arche
#import { os term fmt }

#foreign {
    abs :: proc(int)(int)
    // …other FFI decls
}
```

A banner region runs to end of file, so a file's banner is naturally its only region of that
kind; the error is mainly hit by a stray second block, or a block plus a later banner.
