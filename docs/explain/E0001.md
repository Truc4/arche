# E0001 — undefined symbol

The compiler couldn't resolve a name in expression position. It isn't a
variable in scope, a known function, an archetype, or a constant.

## Common causes

- **Typo**: `prnt(...)` instead of `print(...)`.
- **Out-of-scope reference**: the name was bound in a different proc or block
  and isn't visible here.
- **Missing `#import`**: the symbol lives in a module that hasn't been imported.
- **Defined after first use**: arche names are visible inside the file but the
  order matters in certain contexts (e.g. group members must be declared before
  the group — see E0055).

## Example

```text
entry :: system {
  x := unknown_thing;  // E0001: Undefined symbol 'unknown_thing'
}
#run entry
```

## Fix

Spell the name correctly, bring the binding into scope, or `#import` the module
that exports it. If the name comes from an `#import foo;`, check that the
module file exists and contains the expected declaration.
