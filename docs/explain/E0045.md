# E0045: component_redefined

A component — a nominal type used as an archetype field — must be **defined exactly once**
across the whole program. This error fires when the same name is defined a second time with
the *same* backing type (a different backing is the separate [E0010]).

In Arche's ECS model a component name is global: systems match it by bare name across every
archetype that carries it, so it has one canonical type. Defining it twice is redundant at
best and a latent inconsistency at worst.

```arche
// error: `pos` is defined twice (even though both say `float`)
Particle :: arche { pos :: float, vel :: float }
Body     :: arche { pos :: float, mass :: float }
```

Fix: define each component once — inline in one archetype, or at top level — and reference
it by **bare name** everywhere else:

```arche
Particle :: arche { pos :: float, vel :: float }  // defines pos, vel
Body     :: arche { pos, mass :: float }          // references pos; defines mass
```

or hoist shared components to the top level:

```arche
pos  :: float
vel  :: float
mass :: float
Particle :: arche { pos, vel }
Body     :: arche { pos, mass }
```

Related: **E0010** (redefined with a *different* backing), **E0044** (the same component
listed twice *within one* archetype).
