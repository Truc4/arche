# E0048 — `reduce`/`scan` first argument must be a monoid operator

`reduce` and `scan` fold a column over a **monoid** — an associative operator with an identity. The
operator is the first argument, and it must be one of a fixed built-in set: `+`, `*`, `min`, `max`. It is
not an arbitrary expression or function.

```text
// not:
total := reduce(foo, N.col);   // foo is not a monoid operator
total := reduce(5, N.col);     // a literal is not an operator
```

## Instead

```text
total := reduce(+, N.col);     // sum   (identity 0)
prod  := reduce(*, N.col);     // product (identity 1)
lo    := reduce(min, N.col);   // minimum (identity +inf)
hi    := reduce(max, N.col);   // maximum (identity -inf)
```

## Why

A fold is only well-defined — and only safely reorderable / parallelizable — if the operator is
**associative** and has a known **identity** (the result of folding zero elements). The compiler can only
guarantee that for a closed set it understands, so the operator is a fixed keyword position, not a value.
User-defined monoids (an associative `func` plus a declared identity) are a planned extension; until then,
use one of the four built-ins.
