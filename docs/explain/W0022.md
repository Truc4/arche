# W0022 — exported mutable global

A top-level **mutable** global on the **exported surface** is reported. In a
data-oriented program, shared mutable state belongs in a **pool** — the world's
columnar storage that systems transform in bulk — not in a loose scalar or buffer
that any module can reach and rewrite with no call edge between them.

Unlike most lints, **W0022 is an error by default**. It is still a lint (so it's
tunable — see below), but the default severity is `error`: an exported mutable
global fails the build unless you opt out.

## What trips it

A top-level **mutable** binding whose visibility is **exported** (no `#module` /
`#file` banner narrowing it):

```arche
count := 0;            // W0022 — exported mutable scalar
buf: [64]char;         // W0022 — exported mutable buffer
```

Mutable storage is the `=` side of the binding axis (`name := v`, `name : T = v`,
`name : [N]T`). Immutable definitions (the `::` / `: T :` consts) are not mutable
state and are never flagged.

## Why exported is the dangerous case

The hazard is cross-module action-at-a-distance: one module mutating a global that
another module reads, with no argument or return value recording the dependency.
Arche's premise is explicit, predictable data flow, so the shared-mutable case is
pushed toward the two honest homes — a pool (structured world state) or a private
binding (an implementation detail behind a proc).

## How to fix or opt out

- **Put it in a pool** — the blessed home for global mutable state:
  ```arche
  hp :: int;
  Player :: arche { hp }
  [1]Player(1) { hp: 100 }
  ```
- **Make it private** — `#module` (unit-private) or `#file` (file-private). This
  applies in a **library** (an imported module); in a single-file **binary** the
  bands are moot (there are no importers), so use one of the other opt-outs there.
- **Use an immutable const** if it never changes: `MAX :: 100;` (exported, fine).
- **Relax the lint** with `--exported-mutable=<level>`:
  | Level | Effect |
  |-------|--------|
  | `error` | default — fail the build |
  | `warn`  | report but build succeeds |
  | `allow` | silence entirely |

  ```sh
  arche build --exported-mutable=warn  prog.arche
  arche build --exported-mutable=allow prog.arche
  ```

## Not flagged

- Immutable `::` value consts (exported or not) — not mutable state.
- **Pools** (`[N]Shape`) — the legitimate global mutable state.
- Mutable globals narrowed with `#module` / `#file` (in a library).
- Decls from a **dependency** (stdlib / core).
