# W0014 — unused static / const

A top-level `static` (scalar or array global) was declared but is never
referenced anywhere in the program. Usually dead code left after a refactor.

The visibility model is the same as [W0013](W0013.md): a **public** decl (no
`#module`/`#file` banner) in an imported module is API and is kept; a **private**
one that is referenced nowhere is dead. Dependencies (stdlib/core) are never
linted. In a **binary** entry unit (defines `main`) any unreferenced static is
dead; a **library** entry unit (no `main`) flags nothing.

## Example

```arche
dead_g: int = 9   // W0014: never referenced

entry :: system {
  fmt.printf("hi\n");
}
#run entry
```

## How to silence

- **Use it** — or delete it if it's genuinely dead.
- **Prefix with `_`**: `_scratch: int = 0`.
- **`@allow(unused_static_const)`** (or `@allow(dead_code)`) on the decl.

## Not flagged

- Statics referenced anywhere — in code, an initializer, or an array size.
- Public module statics, dependency (stdlib/core) statics, library-entry statics.
- Pool / archetype storage (its lifetime is the ECS world's, not a plain global).
- Names starting with `_`.
