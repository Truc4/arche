# W0028 — a proc cannot call another proc

In the flat effect model, the three callable kinds do not overlap: a **`func`** builds effects as values,
a **`proc`** is a flat effect *leaf* that runs them, and a **`system`** composes procs. The load-bearing
rule is that a `proc` never calls another `proc`.

The point is to keep the effect graph **depth-1**. If procs could call procs, effects would nest into a
call stack — exactly the depth that makes effects hard to read, test, and reason about. Instead, anything
you would want to *share* between two procs is factored into a **`func` that builds an `Eff` value**; both
procs run that one value. Reuse lives entirely in the value world, so a proc never *needs* to call a proc.

Permitted callees inside a proc body:

- **`extern`** (`#foreign`) — the genuine primitive, the leaf an `Eff` wraps.
- **`func`** — pure value code, including a func that builds an `Eff`.
- **`map`** — a data-parallel kernel.

W0028 is the *local* rule "a proc body may not directly call another proc," which yields the *transitive*
ban for free: if no proc calls a proc directly, none reaches one indirectly. It is stronger than E0050
(`action_in_expression`), which only catches a proc call *nested in an expression* — W0028 also catches the
bare statement form `other_proc();`.

## Severity

W0028 is a **warning by default**. The current stdlib and apps still call procs from procs (the `Eff`
convenience layer that lets them stop is being built), so flipping it to an error now would break essentially
all code. Tune it with:

- `--proc-leaf=warn` — the default (report, don't fail).
- `--proc-leaf=error` — fail the build (the eventual end state).
- `--proc-leaf=allow` — silence it entirely.

Or suppress a single decl with `@allow(proc_calls_proc)`.

## Example

```arche
// BAD: a proc factoring a shared effectful step into a sub-proc
log_line :: proc(fd: int) { write_line(fd, "x"); }   // write_line is a proc → W0028
emit :: proc(fd: int) { log_line(fd); }              // proc → proc → W0028

// GOOD: the shared step is a func that builds an Eff; each proc runs it
line :: func(fd: int, s: []char) -> Eff(int, int) { return fwrite(fd, s); }  // builds, runs nothing
emit :: proc(fd: int) { line(fd, "x")(n:, err:); }   // proc runs the Eff a func built — depth-1
```
