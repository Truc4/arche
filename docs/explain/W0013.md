# W0013 — unused function

A `func` was declared but is never reachable. Usually it's dead code left after a
refactor; sometimes it's a typo (the call spells the name differently).

The model follows Go (unexported) / Rust (`pub`), keyed on arche's region-based
visibility — not a `pub` keyword:

- The **entry unit** (the file you compile) is a **binary** when it declares a
  `#run` schedule: a closed world, so any unreachable decl is dead. With no
  `#run` it's a **library** and its public surface is API — nothing is flagged.
- In an **imported module**, a **public** decl (no `#module`/`#file` banner) is
  exported API and is kept; a **private** decl (under `#module` or `#file`) that
  is unreachable is dead and reported.
- **Dependencies** — the bundled stdlib and core — are never linted.

Roots are the systems/maps the `#run` schedule dispatches (and the
foreign/exported surface); reachability into `func`s is transitive and id-keyed,
so a function reached only by other dead functions is itself dead, and an
unreachable cluster (including mutually-recursive ones) is reported.

## Example

```arche
helper :: func(a: int) -> int {   // never called → W0013
  return a + 1;
}

entry :: system {
  fmt.printf("hi\n");
}
#run entry
```

## How to silence

- **Call it** — or delete it if it's genuinely dead.
- **Prefix with `_`** to keep a named-but-unused function: `_draft :: func() -> int { return 0; }`.
- **`@allow(unused_function)`** (or `@allow(dead_code)`) on the decl:
  ```arche
  @allow(unused_function)
  scaffold :: func() -> int {
    return 0;
  }
  ```

## Not flagged

- The `#run` schedule's systems/maps and everything they reach.
- `#foreign` procs — a C-ABI surface called from outside.
- **Public** module decls (no `#module`/`#file` banner) — exported API.
- Everything in a **library** entry unit (a file with no `#run`).
- Decls from a **dependency** (stdlib / core).
- `@drop` destructors — invoked by the compiler's RAII path, never syntactically.
- Members of an overload group that is used (per-member dead detection inside a
  used group is intentionally not done).
- Names starting with `_`.
- `@allow(unused_function)` or `@allow(dead_code)` on the decl.
