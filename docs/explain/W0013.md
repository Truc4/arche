# W0013 — unused function

A top-level `func` or `proc` was declared but is never reachable from a
program entry point. Usually it's dead code left after a refactor; sometimes
it's a typo (the call spells the name differently).

Reachability is transitive: a function called only by other dead functions is
itself dead, so a whole unreachable cluster (including mutually-recursive ones)
is reported.

## Example

```arche
helper :: func(a: int) -> int {   // never called → W0013
  return a + 1;
}

main :: proc() {
  fmt.print("hi");
}
```

## How to silence

- **Call it** — or delete it if it's genuinely dead.
- **Prefix with `_`** to keep a named-but-unused function: `_draft :: proc() {}`.
- **`@allow(unused_function)`** (or `@allow(dead_code)`) on the decl:
  ```arche
  @allow(unused_function)
  scaffold :: proc() {
  }
  ```

## Not flagged

- `main` — the entry point.
- `extern` / `#foreign` procs — a C-ABI surface called from outside.
- Decls imported from a module (their qualified name carries a `.`); a module's
  API is reachable through qualified access, not local calls.
- A function passed by name as a compile-time callback argument counts as used.
- Members of an overload group that is used (per-member dead detection inside a
  used group is intentionally not done).
- Names starting with `_`.
- The compiler's core prelude (e.g. `streq`).
