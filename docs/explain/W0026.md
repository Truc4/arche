# W0026 — large stack array

A **local fixed-size array** (`name: [N]T`) whose storage exceeds **1 KB** is
reported. A sized array is a *value*: it owns its storage, and a local one lives on
the call stack. A large one inflates the frame — and since arche removed global
top-level arrays, the easy workaround of "move the big buffer into `main`" just
relocates the bulk onto the stack instead of giving it a real home.

This is an ordinary lint: **warning by default**, tunable (see below).

## What trips it

A local `[N]T` whose byte size (`N × sizeof(T)`, summed across dimensions) is at
least 1024:

```arche
main :: proc() {
  buf: [65536]char;   // W0026 — 64 KB on the stack
  xs:  [2000]float;   // W0026 — 8 KB on the stack
  line: [256]char;    // fine — 256 bytes, under the threshold
}
```

Element widths follow codegen: `char`/`bool` = 1, `int`/`float` = 4, a handle or
opaque cell = 8, width-ints by their declared width.

## Not flagged

- **Slices** (`[]T`) — a borrowed view (pointer + length), not owned storage.
- **Parameters** — a borrow of the caller's storage, not a frame allocation.
- Arrays **under** the 1 KB threshold.
- Element types whose size can't be determined (opaque nominals) — never sized, so
  never tripped.

## How to fix or opt out

- **Use a single-type archetype pool** — the blessed home for bulk data. A pool is
  static, columnar storage, not a stack value; slice its column to get a `[]T` buffer:
  ```arche
  b :: char;
  Buf :: arche { b }
  [65536]Buf;          // a 64 KB byte buffer, statically allocated

  // a []char view over the pool's storage — pass it where a buffer is wanted:
  io.read_chunk(fd, Buf.b[0:Buf.capacity], 65536)(chunk:);
  ```
- **Make it a `#module`-private global** if it needs a slice-shaped (`[]T`) API and
  a single shared instance is acceptable (still allowed; not an exported mutable
  global, so no [W0022](W0022.md)):
  ```arche
  #module
  buf: [65536]char;    // module-private, program-lifetime storage
  ```
- **Shrink it** if the size was incidental.
- **Opt out per declaration** with `@allow(large_stack_array)` on the enclosing
  proc/func when a large frame is genuinely intended.
- **Tune the lint** on the command line:
  | Flag | Effect |
  |------|--------|
  | (default) | warning |
  | `-Wno-large-stack-array` | silence entirely |
  | `-Werror=large-stack-array` (or bare `-Werror`) | promote to a hard error |

  ```sh
  arche check -Wno-large-stack-array prog.arche
  arche check -Werror=large-stack-array prog.arche
  ```
